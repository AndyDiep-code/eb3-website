# Locators Output Format

Specification for the locators.md output file generated by the locators capture skill.

## File Location

```
./test-tasks/playwright/{task-id}/locators/{PageName}/locators.md
```

---

## ⛔ CRITICAL: Locator Selection Rules

**Follow this priority order for PRIMARY LOCATOR. Fallback Locator is always XPath — see below.**

| Priority | Method | When to Use |
|----------|--------|-------------|
| 1 | `getByRole(role, { name })` | Buttons, inputs, links, headings — follow W3C ARIA |
| 2 | `getByLabel('label text')` | Form fields with associated label |
| 3 | `getByPlaceholder('placeholder')` | Inputs with placeholder text |
| 4 | `getByText('visible text')` | div, span, paragraph (non-interactive) |
| 5 | `getByAltText('alt text')` | Images |
| 6 | `getByTestId('data-testid')` | Elements with explicit data-testid attributes |
| 7 | `locator('#id')` or `locator('[attr=value]')` | Stable HTML IDs or unique attributes |

### ⛔ Fallback Locator — ALWAYS XPath (REQUIRED)

**The Fallback Locator column MUST always contain an XPath. Never use `—` or leave it empty.**

Generate XPath from ARIA snapshot data using these rules:

| ARIA Role | XPath Template |
|-----------|----------------|
| `button` (with name) | `locator('xpath=//button[normalize-space()="{name}"]')` |
| `button` (no name / icon-only) | `locator('xpath=(//*[@role="button"])[{nth}]')` |
| `textbox` (with aria-label) | `locator('xpath=//input[@aria-label="{name}"]')` |
| `textbox` (with placeholder) | `locator('xpath=//input[@placeholder="{placeholder}"]')` |
| `combobox` | `locator('xpath=//input[@aria-label="{name}"]')` |
| `link` | `locator('xpath=//a[normalize-space()="{name}"]')` |
| `heading` | `locator('xpath=//*[@role="heading"][normalize-space()="{name}"]')` |
| `img` | `locator('xpath=//img[@alt="{name}"]')` |
| `checkbox` | `locator('xpath=//input[@type="checkbox"][@aria-label="{name}"]')` |
| `option` | `locator('xpath=//*[@role="option"][normalize-space()="{name}"]')` |
| `columnheader` | `locator('xpath=//th[normalize-space()="{name}"]')` |
| `listbox` | `locator('xpath=//*[@role="listbox"]')` |
| `generic` / `paragraph` | `locator('xpath=//*[normalize-space()="{text}"]')` |
| Any element (positional fallback) | `locator('xpath=(//{tag})[{nth}]')` |

**XPath rules:**
- Always use `normalize-space()` for text matching (handles whitespace)
- Prefer `@aria-label` over text content for form fields
- For elements with no unique name: use positional XPath e.g. `(//button)[3]`
- Never use `[ref=eXX]` in XPath
- Double-quote attribute values inside single-quoted xpath string

### ❌ FORBIDDEN — NEVER use these in locators.md or automation code

```typescript
// ❌ WRONG — [ref=eXX] is an internal playwright-cli snapshot reference.
//            It is NOT a DOM attribute. page.locator('[ref=e14]') will ALWAYS timeout.
page.locator('[ref=e14]')
page.locator('[ref=e11]')

// ❌ WRONG — Generic CSS without stable anchor
page.locator('div > button')
page.locator('.btn')   // class names change frequently
```

### ✅ CORRECT examples

```typescript
// ✅ getByRole — most reliable for buttons, inputs, links
page.getByRole('button', { name: 'Login with Microsoft' })
page.getByRole('textbox', { name: 'Email' })
page.getByRole('link', { name: 'Forgot my password' })

// ✅ getByLabel — for labeled form fields
page.getByLabel('Email address')
page.getByLabel('Password')

// ✅ getByPlaceholder — for inputs with placeholder
page.getByPlaceholder('Email, phone, or Skype')

// ✅ Stable ID — only if the ID is not dynamically generated
page.locator('#i0116')      // Microsoft email input (stable)
page.locator('#i0118')      // Microsoft password input (stable)
page.locator('#idSIButton9') // Microsoft submit button (stable)

// ✅ XPath last resort — prefer role-based above
page.locator('//button[normalize-space()="Sign in"]')
```

```markdown
## Navigation Elements

| Name | Role | Primary Locator | Fallback Locator | Description |
|------|------|----------------|------------------|-------------|
| SiteLogo | link | `getByRole('link', { name: 'Home' })` | `locator('xpath=//a[normalize-space()="Home"]')` | Main site logo |
| SearchBox | textbox | `getByPlaceholder('Search')` | `locator('xpath=//input[@placeholder="Search"]')` | Search input |
| LoginButton | button | `getByRole('button', { name: 'Login' })` | `locator('xpath=//button[normalize-space()="Login"]')` | Login button |

## Form Elements

| Name | Role | Primary Locator | Fallback Locator | Description |
|------|------|----------------|------------------|-------------|
| EmailField | textbox | `getByLabel('Email')` | `locator('xpath=//input[@aria-label="Email"]')` | Email input |
| PasswordField | textbox | `getByLabel('Password')` | `locator('xpath=//input[@aria-label="Password"]')` | Password input |
| SubmitButton | button | `getByRole('button', { name: 'Sign in' })` | `locator('xpath=//button[normalize-space()="Sign in"]')` | Submit button |
```

---

## Column Definitions

| Column | Description |
|--------|-------------|
| **Name** | Meaningful camelCase name (e.g., `loginButton`, `emailInput`) |
| **Role** | ARIA role from the element (button, textbox, link, checkbox, etc.) |
| **Primary Locator** | Best locator — use `getByRole`, `getByLabel`, `getByPlaceholder` first |
| **Fallback Locator** | XPath selector — **ALWAYS required**, **ALWAYS XPath format** (`locator('xpath=...')`), NEVER `—` |
| **Description** | What the element does in plain language |

> ⚠️ The `Ref` column (e.g., `e11`, `e14`) from ARIA snapshots is intentionally **excluded** from the locator table.
> Refs are internal playwright-cli identifiers, not DOM attributes. They CANNOT be used in Playwright automation code.

---

## Naming Conventions

- **camelCase**: `loginButton`, `emailInput`, `submitButton`
- **Descriptive**: Indicate what the element does
- **Unique**: No duplicates on the same page

## Framework-Specific Notes

### Flutter Web (CanvasKit renderer)

Flutter renders on canvas. Elements are only accessible via the semantic overlay (requires `--force-renderer-accessibility` in `playwright.config.ts`).

```typescript
// ✅ Best for Flutter — role + accessible name from semantic overlay
page.getByRole('button', { name: 'Login with Microsoft' })
page.getByText('Welcome to Enoverse!')

// ✅ Flutter semantic element selector (fallback)
page.locator('flt-semantics[aria-label="Login with Microsoft"]')

// ❌ NEVER — Flutter refs are useless in tests
page.locator('[ref=e11]')
```

### Standard HTML (e.g., Microsoft OAuth, regular pages)

```typescript
// ✅ Stable IDs (prefer when IDs are not dynamic)
page.locator('#i0116')         // email input
page.locator('#idSIButton9')   // submit button

// ✅ Role-based
page.getByRole('button', { name: 'Sign in' })
page.getByPlaceholder('Email, phone, or Skype')
```

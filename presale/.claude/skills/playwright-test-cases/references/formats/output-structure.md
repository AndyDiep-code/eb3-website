# Output Structure — Test Cases

Canonical directory structure for generated test cases (draft, task-scoped).

Mirrors the task workspace structure. Promoted to repo via `test-cases-promote`.

## Directory Layout

```
test-tasks/playwright/{task-id}/test-cases/
│
├── README.md                        ← index of all test cases
│
├── pages/                           ← mirrors tests/playwright/docs/pages/
│   └── {PageName}/                  ← one folder per captured page
│       ├── test-cases.md            ← test cases in mandatory format
│       └── test-data.json           ← test data fixtures
│
└── features/                        ← mirrors tests/playwright/docs/features/ (Step 6.5 only)
    └── {feature-name}/              ← one folder per feature (if feature docs exist)
        ├── test-cases.md            ← TC-F-001, TC-F-002...
        └── test-data.json
```

> **API test cases** (`api/` folder) are generated by the separate `api:test-cases` command, not by this skill.

## Folder Rules

| Folder | When to create |
|--------|---------------|
| `pages/{PageName}/` | Always — one per page in `locators/` |
| `features/{feature-name}/` | Only if Step 6.5 preconditions met (feature docs in `docs/features/` exist) |

## Mirror Relationship

```
tests/playwright/docs/pages/{PageName}/   →  test-tasks/playwright/{task-id}/test-cases/pages/{PageName}/
  business-context.md                          test-cases.md
  user-flows.md                                test-data.json
  data-model.md

tests/playwright/docs/features/{Name}/    →  test-tasks/playwright/{task-id}/test-cases/features/{Name}/
  overview.md                                  test-cases.md
  user-journeys.md                             test-data.json
```

## File Rules

### `test-cases.md`
Must follow `references/formats/ui-test-case-template.md`.

ID conventions:
- Page tests: `TC-HP-001`, `TC-NEG-001`, `TC-EDGE-001`, `TC-VAL-001`
- Feature flows: `TC-F-001`, `TC-F-002`
- API tests: `TC-API-001`

### `test-data.json`
Must follow `references/formats/test-data-schema.md`.

### `README.md`
Summary table — all pages/features/api, case counts, priority breakdown. Updated every run.

## Example

```
test-tasks/playwright/{task-id}/test-cases/
├── README.md
├── pages/
│   ├── AttendancePage/
│   │   ├── test-cases.md
│   │   └── test-data.json
│   ├── CreateNewRequestRemote/
│   │   ├── test-cases.md
│   │   └── test-data.json
│   └── ...
├── features/
│   └── attendance-remote-request/
│       ├── test-cases.md
│       └── test-data.json
└── (no api/ — generated by api:test-cases command separately)
```

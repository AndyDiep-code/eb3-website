# ARIA Snapshot Format

Specification for the ARIA snapshot YAML file generated by Playwright's snapshot command.

## File Location

```
./test-tasks/playwright/{task-id}/locators/{PageName}/aria-snapshot.yaml
```

## YAML Structure

```yaml
- generic [ref=e1]:
  - heading "Login" [level=1] [ref=e2]
  - textbox [ref=e3]:
    - /placeholder: "Email"
    - /value: ""
  - textbox [ref=e4]:
    - /placeholder: "Password"
    - /value: ""
  - checkbox [ref=e5]:
    - /checked: false
  - button "Login" [ref=e6]:
    - /disabled: false
  - link "Sign Up" [ref=e7]:
    - /href: "/signup"
  - link "Forgot Password?" [ref=e8]:
    - /href: "/reset"
```

## Element Structure

Each element in the ARIA tree has:

```yaml
- {role} "{text}" [ref=eXX]:
  - /{property}: {value}
  - /{property}: {value}
```

### Components

| Component | Description |
|-----------|-------------|
| `role` | ARIA role (button, textbox, link, heading, etc.) |
| `text` | Visible text content (if any) |
| `[ref=eXX]` | Unique reference ID for Playwright interaction |
| `/{property}` | ARIA property (placeholder, value, checked, disabled, href, etc.) |
| `{value}` | Property value (string, boolean, number) |

## Common ARIA Roles

| Role | Description | Example |
|------|-------------|---------|
| `button` | Clickable button | `<button>Click me</button>` |
| `textbox` | Text input field | `<input type="text">` |
| `checkbox` | Checkbox input | `<input type="checkbox">` |
| `radio` | Radio button | `<input type="radio">` |
| `link` | Hyperlink | `<a href="...">Link</a>` |
| `heading` | Heading | `<h1>Title</h1>` |
| `list` | List container | `<ul>` or `<ol>` |
| `listitem` | List item | `<li>` |
| `menuitem` | Menu item | Menu option |
| `dialog` | Modal dialog | `<dialog>` |
| `generic` | Generic container | `<div>` |
| `region` | Content region | `<section>` |
| `navigation` | Navigation area | `<nav>` |
| `main` | Main content | `<main>` |
| `complementary` | Sidebar/complementary | `<aside>` |

## Common ARIA Properties

| Property | Type | Description |
|----------|------|-------------|
| `/placeholder` | string | Placeholder text for input |
| `/value` | string | Current value of input |
| `/checked` | boolean | Checkbox/radio checked state |
| `/disabled` | boolean | Element disabled state |
| `/href` | string | Link URL |
| `/level` | number | Heading level (1-6) |
| `/expanded` | boolean | Expandable element state |
| `/selected` | boolean | Selected state |
| `/readonly` | boolean | Read-only state |
| `/required` | boolean | Required field |
| `/label` | string | Associated label text |

## Nesting

Elements can be nested to show hierarchy:

```yaml
- generic [ref=e1]:
  - heading "Login Form" [level=1] [ref=e2]
  - generic [ref=e3]:
    - textbox [ref=e4]:
      - /placeholder: "Email"
    - textbox [ref=e5]:
      - /placeholder: "Password"
    - button "Login" [ref=e6]
  - link "Forgot Password?" [ref=e7]
```

## Reference IDs

- Format: `eXX` where XX is a number (e1, e2, e3, ..., e246)
- Unique per page
- Used for Playwright interaction
- Assigned by Playwright snapshot command

## ⛔ CRITICAL: How to Use ARIA Snapshot Refs in Playwright Tests

### What `[ref=eXX]` is

`[ref=e14]`, `[ref=e11]`, etc. are **internal playwright-cli snapshot identifiers**.
They are assigned by the `playwright-cli snapshot` command to label elements in the YAML output.

**They are NOT DOM attributes.** The browser's HTML does NOT contain `ref="e14"` on any element.

### ❌ WRONG — will ALWAYS timeout in Playwright tests

```typescript
// ❌ NEVER DO THIS — [ref=eXX] is not a DOM attribute
await page.locator('[ref=e14]').click();
await page.locator('[ref=e11]').fill('user@example.com');
await page.locator('[ref=e9]').textContent();

// These will timeout with:
// "TimeoutError: locator.click: Timeout 10000ms exceeded.
//  Call log: waiting for locator('[ref=e14]')"
```

### ✅ CORRECT — use role, label, placeholder, text, or stable selectors

**Priority order (Playwright official recommendation):**

```typescript
// 1. getByRole — BEST for interactive elements (buttons, inputs, links)
await page.getByRole('button', { name: 'Sign In' }).click();
await page.getByRole('textbox', { name: 'Email address' }).fill('user@example.com');
await page.getByRole('link', { name: 'Forgot password?' }).click();

// 2. getByLabel — for form fields with labels
await page.getByLabel('Email').fill('user@example.com');
await page.getByLabel('Password').fill('secret');

// 3. getByPlaceholder — for inputs with placeholder text
await page.getByPlaceholder('Email, phone, or Skype').fill('user@example.com');

// 4. getByText — for non-interactive text elements
await page.getByText('Welcome to Enoverse!').isVisible();

// 5. Stable ID/attribute — when element has a reliable ID
await page.locator('#i0116').fill('user@example.com');
await page.locator('input[name="loginfmt"]').fill('user@example.com');

// 6. XPath — last resort only
await page.locator('//button[normalize-space()="Sign In"]').click();
```

### How to map ARIA snapshot → Playwright locator

When you see this in `aria-snapshot.yaml`:
```yaml
- button "Sign In" [ref=e14]
- textbox [ref=e11]:
  - /placeholder: "Email address"
```

Convert to Playwright like this:
```typescript
// button "Sign In" [ref=e14]  →  getByRole('button', { name: 'Sign In' })
await page.getByRole('button', { name: 'Sign In' }).click();

// textbox [ref=e11] with /placeholder "Email address"  →  getByPlaceholder
await page.getByPlaceholder('Email address').fill('user@example.com');
// OR getByRole with name matching the label:
await page.getByRole('textbox', { name: 'Email address' }).fill('user@example.com');
```

## Best Practices

1. **NEVER use `[ref=eXX]` in automation code** — it will timeout every time
2. **Read the ARIA snapshot for context** — use role + accessible name to build `getByRole`
3. **Use `/placeholder` property** — maps to `getByPlaceholder`
4. **Use `/label` property** — maps to `getByLabel`
5. **Use element text** — `button "Sign In"` → `getByRole('button', { name: 'Sign In' })`
6. **Keep snapshot as documentation only** — for understanding page structure, not for copy-paste selectors

## Troubleshooting

### Missing Elements

If an element doesn't appear in snapshot:
- Element might be hidden (display: none, visibility: hidden)
- Element might be outside viewport
- Element might be rendered dynamically (check after interaction)
- Element might not have proper ARIA role

### Incorrect Refs

If refs change between snapshots:
- Page structure changed
- Elements added/removed
- DOM reordered
- Take new snapshot after changes

### Property Not Found

If expected property missing:
- Property might not be set on element
- Use browser DevTools to inspect element
- Check ARIA specification for valid properties

# Network Capture Format

Complete specification for network capture data generated by the locators capture skill.

## Output File Location

```
./test-tasks/playwright/{task-id}/locators/
├── {PageName}/network-calls.json          (SIMPLE MODE, SESSION MODE)
└── network-calls.json                     (INTERACTIVE MODE - aggregated)
```

## JSON Structure

```json
{
  "capturedAt": "2026-04-02T10:30:00.000Z",
  "pageUrl": "https://example.com/dashboard",
  "networkCalls": [
    {
      "sequence": 1,
      "method": "GET",
      "url": "https://api.example.com/users",
      "status": 200,
      "statusText": "OK",
      "requestHeaders": {
        "authorization": "[REDACTED]",
        "accept": "application/json"
      },
      "requestBody": null,
      "responseHeaders": {
        "content-type": "application/json",
        "x-request-id": "req-12345"
      },
      "responseBody": {
        "data": [
          { "id": 1, "name": "User 1" }
        ]
      },
      "duration": 234,
      "timestamp": "2026-04-02T10:30:00.000Z",
      "error": null
    }
  ],
  "summary": {
    "totalRequests": 1,
    "successfulRequests": 1,
    "failedRequests": 0,
    "averageDuration": 234,
    "endpoints": {
      "GET /api/users": { "count": 1, "avgDuration": 234 }
    }
  },
  "sequences": [
    {
      "flow": "Flow_1_api_example_com",
      "calls": [
        {
          "order": 1,
          "method": "GET",
          "url": "https://api.example.com/users",
          "status": 200
        }
      ]
    }
  ]
}
```

## Field Descriptions

### Top Level

| Field | Type | Description |
|-------|------|-------------|
| `capturedAt` | string (ISO 8601) | When capture listener was initialized |
| `pageUrl` | string | URL of page at time of capture |
| `networkCalls` | array | Array of captured network calls |
| `summary` | object | Aggregated metrics and enhancements |
| `sequences` | array | Call sequences grouped by domain |

### Network Call Record

| Field | Type | Description |
|-------|------|-------------|
| `sequence` | number | Auto-increment, 1-based order |
| `method` | string | HTTP method (GET, POST, PUT, PATCH, DELETE, HEAD, OPTIONS) |
| `url` | string | Full absolute URL |
| `status` | number | HTTP response status code (200-599) |
| `statusText` | string | Status text (e.g., "OK", "Created", "Not Found") |
| `requestHeaders` | object | Request headers (sensitive values redacted) |
| `requestBody` | object\|string\|null | Parsed JSON or raw text; null for bodyless methods |
| `responseHeaders` | object | Response headers (sensitive values redacted) |
| `responseBody` | object\|string\|null | Parsed JSON or raw text; null on error |
| `duration` | number | Response time in milliseconds |
| `timestamp` | string (ISO 8601) | When request was initiated |
| `error` | string\|null | Capture error (not HTTP error) |

### Summary Metrics

| Field | Type | Description |
|-------|------|-------------|
| `totalRequests` | number | Total captured network calls |
| `successfulRequests` | number | 2xx and 3xx responses |
| `failedRequests` | number | 4xx and 5xx responses |
| `averageDuration` | number | Average response time (ms) |
| `endpoints` | object | API endpoint inventory (Enhancement #1) |

### Endpoint Inventory (Enhancement #1)

```json
{
  "endpoints": {
    "GET /api/users": { "count": 5, "avgDuration": 245 },
    "POST /api/users": { "count": 1, "avgDuration": 520 },
    "GET /api/users/{id}": { "count": 3, "avgDuration": 180 }
  }
}
```

- Normalizes URLs by replacing numeric IDs with `{id}` placeholder
- Removes query parameters and hash fragments
- Groups by method + normalized URL pattern

### Call Sequences (Enhancement #3)

```json
{
  "sequences": [
    {
      "flow": "Flow_1_api_example_com",
      "calls": [
        {
          "order": 1,
          "method": "POST",
          "url": "https://api.example.com/login",
          "status": 200,
          "responseExtracted": ["token", "userId"]
        },
        {
          "order": 2,
          "method": "GET",
          "url": "https://api.example.com/users/{id}",
          "status": 200,
          "dependsOnFields": ["id"]
        }
      ]
    }
  ]
}
```

- Groups calls by domain into separate flow sequences
- Auto-numbers calls in sequence order
- Detects extracted fields from responses (token, userId, sessionId, id)
- Marks calls that depend on previously extracted fields

## Automatic Filtering

Network capture automatically excludes:
- Static assets (.png, .jpg, .gif, .css, .woff, .ttf, etc.)
- Minified bundles and source maps
- Favicon and health check endpoints
- 1xx informational responses
- Invalid status codes (null, 0, >= 600)

Includes:
- All API calls (2xx, 3xx, 4xx, 5xx)
- JSON and text responses
- Request/response bodies (capped at 100KB)

## Sensitive Data Handling

### Redacted Headers

The following headers are automatically redacted as `[REDACTED]`:
- Authorization
- Cookie
- Set-Cookie
- X-API-Key
- X-Auth-Token
- X-Access-Token
- X-Secret-Key
- API-Key
- X-CSRF-Token

### Body Truncation

Request and response bodies are independently capped at 100KB:
- If body exceeds 100KB: truncated and appended with `\n...[TRUNCATED - body exceeded 100KB limit]`
- Prevents excessive disk usage and memory overhead
- Both bodies capped independently

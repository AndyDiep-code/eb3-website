# Test Case Output Format

Canonical format for `test-cases.md` generated by this skill.
Compatible with Phase 3 (`playwright-api-supertest`) as input.

---

## File Header

```markdown
# API Test Cases: {TASK_ID}

| Field | Value |
|-------|-------|
| Generated | {YYYY-MM-DD} |
| Source | OpenAPI spec ({BASE_URL}/api-json) |
| Server | {BASE_URL} |
| Operations | {N} |
| Total cases | {N} |
```

---

## Section Structure

Group by module/tag, then by test type within each module:

```
# API Test Cases: {task-id}

## Module: Auth

TC-API-001 ...
TC-API-002 ...

TC-FLOW-001 ...

### RBAC Tests
TC-RBAC-001 ...

## Module: Resources
...

## Module: Bookings
...

## Module: Users
...
```

## Per-test Format

### Contract / Unit test

```markdown
### TC-API-{NUM}: {METHOD} {PATH} — {brief title}

**Type:** Happy Path | Negative | Validation | Edge
**Module:** {tag}
**Auth:** None | Bearer (any) | Bearer (admin)

**Request:**
- Method: {METHOD}
- Path: `{path with :params noted}`
- Body: `{"field": "value"}`
- Headers: `Authorization: Bearer {ADMIN_TOKEN|USER_TOKEN|none}`

**Expected:**
- Status: {code}
- Response contains: `{"key": "expected_value"}`

**Notes:** {optional — why this case matters}
```

### Multi-step Flow test

```markdown
### TC-FLOW-{NUM}: {Flow Name}

**Type:** Flow / Integration
**Actors:** {e.g., Admin + User}
**Pre-condition:** {e.g., "Resource #1 exists (seeded)"}

**Steps:**
1. `POST /auth/login` (admin) → extract `access_token` as ADMIN_TOKEN
2. `POST /bookings` (user token) `{"resourceId": 1, ...}` → extract `id` as BOOKING_ID
3. `PUT /bookings/{BOOKING_ID}` (admin token) `{"status": "confirmed"}` → 200
4. `GET /bookings/{BOOKING_ID}` (user token) → verify `status == "confirmed"`

**Expected outcome:** {final state description}
```

### RBAC test

```markdown
### TC-RBAC-{NUM}: {Endpoint} access control

**Type:** RBAC
**Endpoint:** {METHOD} {PATH}

| Scenario | Token | Expected Status | Notes |
|----------|-------|----------------|-------|
| No auth | none | 401 | |
| User on admin-only | USER_TOKEN | 403 | |
| Admin | ADMIN_TOKEN | {expected} | |
| User on own data | USER_TOKEN | {expected} | |
| User on other's data | USER_TOKEN | 403 | cross-user access |
```

## Token Placeholder Convention

Use these exact placeholders in test cases — Phase 3 (supertest) replaces them:

| Placeholder | Meaning |
|-------------|---------|
| `ADMIN_TOKEN` | Bearer token from admin login |
| `USER_TOKEN` | Bearer token from regular user login |
| `RESOURCE_ID` | ID of a seeded/created resource |
| `BOOKING_ID` | ID of a created booking |
| `USER_ID` | ID of a seeded user |

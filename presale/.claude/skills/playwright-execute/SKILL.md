# Playwright Execute Skill

## Arguments

- `--task=<id>` ‚Äî Filter by `@task:<id>` tag (REQUIRED unless `--all`)
- `--page=<Name>` ‚Äî Filter by `@page:<Name>` tag (optional)
- `--case=<TC-XXX>` ‚Äî Filter by `@tc:<TC-XXX>` tag (optional)
- `--all` ‚Äî Run all tests (no task filter)
- `--headed` ‚Äî Show browser window
- `--workers=<N>` ‚Äî Parallel workers (default: 1)
- `--ui` ‚Äî Open Playwright Test UI
- `--debug` ‚Äî Debug mode (headed, 1 worker)

## Core Principle

This skill does NOT simulate test steps manually. It runs real Playwright test scripts via `npx playwright test` and uses AI to analyze the actual output.

## Argument Mapping

Map command arguments to Playwright CLI flags:

| Argument | Playwright Flag |
|----------|----------------|
| `--task=<id>` | `--grep "@task:<id>"` |
| `--page=<Name>` | `--grep "@page:<Name>"` |
| `--case=<TC-XXX>` | `--grep "@tc:<TC-XXX>"` |
| `--all` | (no grep filter) |
| `--headed` | `--headed` |
| `--workers=<N>` | `--workers=<N>` |
| `--ui` | `--ui` |
| `--debug` | `--debug --headed --workers=1` |

Multiple filters combine with AND: `--task=test-login --page=LoginPage` ‚Üí `--grep "@task:test-login.*@page:LoginPage"` (or run separately and intersect)

## Script Location

Generated test scripts live in:
```
./tests/playwright/tests/ui/pages/{PageName}/test-cases.spec.ts
```

Generated by the `automate` command. Always verify they exist before running.

## Operating Rules

0. **If no arguments provided** ‚Üí run Step 0 (interactive gather via `AskUserQuestion`) before proceeding
1. Validate task + confirm test scripts exist before running
2. Run from `./tests/playwright/` directory (where `package.json` is)
3. Always use `--reporter=json,list` to capture machine-readable output
4. Monitor stdout/stderr in real-time ‚Äî log progress as it streams
5. After run: read `playwright-report/results.json` for structured data
6. For each failure: read screenshot path + trace path from the report
7. Deliver AI analysis grouped by failure ‚Äî root cause + concrete fix suggestion
8. Save analysis report to task execution directory
9. If no test scripts found: stop and tell user to run `automate` command first
10. If playwright not installed: stop and tell user to run `init-automation` first
